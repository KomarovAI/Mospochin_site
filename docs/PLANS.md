# Execution Plans

Create a plan only when a task changes many pages, a generator, a schema, backend behavior or deployment.

- Active plans: `docs/exec-plans/active/`.
- Completed plans: `docs/exec-plans/completed/`.
- Do not create a plan for a small text, metadata or single-page edit.
- Plans contain goals, scope, invariants, steps, verification and rollback.
- Plans do not duplicate repository instructions or current project metrics.
