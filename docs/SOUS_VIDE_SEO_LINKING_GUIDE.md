# Sous-vide SEO content and internal-linking guide

## Purpose

This document defines the permanent SEO/content and internal-linking contract for the sous-vide cluster. The machine-readable graph is `data/sous-vide-link-graph.json`; symptom relationships live in `data/sous-vide-fault-taxonomy.json`.

## Google-aligned principles

- Important pages must be reachable through normal crawlable `<a href>` links.
- Anchor text must describe the destination before the user opens it.
- Site structure is inferred primarily from page-to-page linking, not only from URL paths.
- Structured data must match visible page content.
- Similar pages must have independent user value and must not operate as doorway variations.
- FAQ remains visible user content. `FAQPage` parity is maintained, but the project does not promise FAQ rich results because Google normally restricts them to authoritative government and health sites.

## Symptom-page link contract

Every published symptom page links to:

1. the sous-vide cluster hub;
2. the overview page for the same equipment family;
3. the repair page for the same equipment family;
4. at least three related symptom pages from the same equipment family;
5. an optional contextual guide when it helps the user complete the diagnostic path.

Thermostat symptoms must not link to vacuum-packer symptoms merely to satisfy a link-count minimum. Vacuum-packer symptoms follow the same rule in reverse.

The exact relationship list is stored in each taxonomy record as `relatedSymptoms`. The HTML block is generated by:

```bash
npm run generate:sous-vide-seo-layer
```

Do not edit `sections/100-related-symptoms.html` manually without changing taxonomy or the graph contract first.

## Content contract

The diagnostic core must remain page-specific:

- safe self-check;
- decision tree;
- cause matrix;
- explained diagnostic nodes;
- FAQ.

The repair-scope cards are generated from `probableNodeLabels` and `probableNodeDescriptions`. A repeated placeholder such as “the decision is made after measurements” is forbidden because it gives no information about the specific node.

## Hub contract

The cluster hub groups symptom pages by equipment family:

- thermostats and water baths;
- chamber vacuum packers.

Each symptom appears exactly once and uses a target-specific description taken from the taxonomy. The hub is allowed to be flat for crawl discovery, while symptom-to-symptom links provide the semantic subgraph.
