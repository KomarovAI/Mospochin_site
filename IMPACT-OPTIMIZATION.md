# 🚀 РЕАЛЬНО ИМПАКТНАЯ ОПТИМИЗАЦИЯ САЙТА

## Что сделано (ТОЛЬКО импактные изменения)

### ✅ 1. Создан `DATA-INDEX.md` - единый указатель всех данных

**Проблема:** 23 JSON файла в `data/`, непонятно какой за что отвечает.

**Решение:** Создан подробный индекс с таблицами:
- Какой файл за что отвечает
- Что можно менять в каждом файле
- Примеры реальных изменений
- Топ-5 файлов для 90% задач

**Импакт:** 
- Нейронка теперь знает какой файл открывать
- Человек понимает структуру за 2 минуты
- Не нужно гадать "а где тут FAQ?"

**Файл:** `DATA-INDEX.md`

---

### ✅ 2. Создан `AGENTS-EXTENDED.md` - конкретные правила для AI

**Проблема:** `AGENTS.md` слишком общий, нет конкретных примеров.

**Решение:** Создан расширенный файл с:
- ✅ Что МОЖНО делать (с примерами кода)
- ❌ Что НЕЛЬЗЯ делать (с объяснением почему)
- 🔥 Примеры правильных изменений (шаг за шагом)
- 🛡️ Проверка перед коммитом
- 🚨 Что делать если что-то сломалось

**Импакт:**
- Нейронка не сломает сайт случайными изменениями
- Конкретные примеры вместо абстрактных правил
- Чеклист перед каждым изменением

**Файл:** `AGENTS-EXTENDED.md`

---

### ✅ 3. Создан `tools/ai-check.mjs` - автоматическая проверка

**Проблема:** После изменений нужно вручную проверять что сайт работает.

**Решение:** Создан скрипт который автоматически:
1. Проверяет валидность всех JSON файлов
2. Проверяет что все HTML файлы существуют
3. Запускает `validate:site`
4. Проверяет размер файлов
5. Выводит итоговый отчёт

**Импакт:**
- Одна команда: `npm run ai:check`
- Автоматическая проверка вместо ручной
- Сразу видно что сломалось

**Команда:** `npm run ai:check`

---

### ✅ 4. Добавлена команда `ai:check` в `package.json`

**Что делает:** Запускает автоматическую проверку сайта.

**Использование:**
```bash
npm run ai:check
```

**Когда использовать:**
- После каждого изменения JSON файлов
- Перед коммитом
- Если что-то выглядит сломанным
