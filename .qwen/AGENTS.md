
## Правила оптимизации токенов (2026-04-08)

### Размер шага
- **Максимум 1 файл или 5 файлов за шаг.**
- **Каждый шаг:** правка → verify → commit → следующий.
- **Стоп-кран:** если шаг > 3 итераций → останавливаюсь, спрашиваю.

### Суб-агенты
- `code-reviewer` → после каждого изменения кода (ОБЯЗАТЕЛЬНО)
- `testing-expert` → при изменении JS-логики
- `memory-manager` → после завершения фазы

### Спецификации
- Перед изменениями UI — прочитать `.qwen/specs/ui-components.md`
- Следовать правилам из spec, не импровизировать

### Деплой
- Всегда использовать `deploy.sh` или `rsync` только изменённых файлов
- Всегда проверять HTTP 200 после деплоя
