## ❌ ЧТО НЕЛЬЗЯ ДЕЛАТЬ (Danger Zone)

### 1. НЕ редактируй HTML напрямую
```html
<!-- ❌ НЕЛЬЗЯ: Менять вёрстку в holodilniki.html -->
<section class="hero">
  <h1>Новый заголовок</h1>  <!-- НЕ ДЕЛАЙ ТАК! -->
</section>
```

**Почему:** HTML файлы огромные (183KB), легко сломать вёрстку. Используй JSON данные вместо этого.

---

### 2. НЕ трогай CSS файлы
```css
/* ❌ НЕЛЬЗЯ: Менять стили в styles.css */
.hero {
  background: red;  /* НЕ ДЕЛАЙ ТАК! */
}
```

**Почему:** CSS используется на всех страницах, одно изменение может сломать весь сайт.

---

### 3. НЕ трогай JavaScript файлы
```javascript
// ❌ НЕЛЬЗЯ: Менять логику в main.js
document.addEventListener('DOMContentLoaded', () => {
  // НЕ ДЕЛАЙ ТАК!
});
```

**Почему:** JS отвечает за формы, аналитику, интерактивность. Одно изменение может сломать заявки.

---

### 4. НЕ удаляй системные файлы
```
❌ НЕЛЬЗЯ: Удалять эти файлы:
- data/household-page-policy.json
- data/restaurant-page-policy.json
- data/site-page-contracts.json
- data/operator-recipes.json
- tools/*.mjs (если не создаёшь новые)
```

**Почему:** Эти файлы определяют структуру сайта и защиту от ошибок.

---

### 5. НЕ меняй структуру JSON
```json
// ❌ НЕЛЬЗЯ: Менять структуру данных
{
  "pages": {
    "bytovaya-holodilniki.html": {
      "newField": "value"  // НЕ ДОБАВЛЯЙ НОВЫЕ ПОЛЯ!
    }
  }
}
```

**Почему:** HTML шаблоны ожидают конкретную структуру. Новые поля не отобразятся.

## 🔥 ПРИМЕРЫ ПРАВИЛЬНЫХ ИЗМЕНЕНИЙ

### Пример 1: Изменить цену на услугу

**Задача:** Изменить цену "Замена компрессора" с 5000 на 4500 рублей

**Правильный способ:**
```bash
# Используй простую команду
npm run ai:price --page=bytovaya-holodilniki.html --service="Замена компрессора" --price=4500
```

**Или вручную:**
1. Открой `data/household-page-slots.json`
2. Найди страницу `bytovaya-holodilniki.html`
3. Найди секцию `prices`
4. Измени цену:
```json
"prices": [
  {
    "service": "Замена компрессора",
    "price": "от 4500 ₽"  // ← было 5000, стало 4500
  }
]
```
5. Запусти валидацию: `npm run validate:site`

---

### Пример 2: Добавить новый FAQ вопрос

**Задача:** Добавить вопрос "Работаете ли в выходные?" с ответом "Да, работаем 24/7"

**Правильный способ:**
```bash
# Используй простую команду
npm run ai:faq-add --page=bytovaya-holodilniki.html \
  --q="Работаете ли в выходные?" \
  --a="Да, работаем 24/7 без выходных и праздников"
```

**Или вручную:**
1. Открой `data/household-page-slots.json`
2. Найди страницу `bytovaya-holodilniki.html`
3. Найди массив `faq`
4. Добавь новый объект:
```json
"faq": [
  {"question": "Сколько стоит ремонт?", "answer": "От 2000 рублей"},
  {"question": "Работаете ли в выходные?", "answer": "Да, работаем 24/7 без выходных и праздников"}
]
```
5. Запусти валидацию: `npm run validate:site`

Перед тем как что-то менять, проверь:

- [ ] Я меняю JSON файл, а не HTML/CSS/JS?

## 🚨 ЕСЛИ ЧТО-ТО СЛОМАЛОСЬ

### Сценарий 1: Ошибки валидации
```bash
npm run validate:site
# Вывод: Error in household-page-slots.json: Invalid JSON
```

**Решение:**
1. Открой файл `data/household-page-slots.json`
2. Проверь JSON синтаксис (запятые, кавычки, скобки)
3. Используй онлайн валидатор: https://jsonlint.com/
4. Исправь ошибку
5. Снова запусти `npm run validate:site`

---

### Сценарий 2: Сайт не отображается правильно
```bash
npm run dev
# Открой http://localhost:3000
# Вывод: Страница выглядит сломанной
```

**Решение:**
1. Откати изменения:
```bash
git diff  # Посмотри что изменилось
git checkout data/household-page-slots.json  # Откати файл
```
2. Проверь что ты не изменил структуру JSON
3. Попробуй снова, но аккуратнее

---

### Сценарий 3: Полная катастрофа
```bash
# Всё сломалось, не знаю что делать
```

**Решение:**
```bash
# Откати все изменения
git reset --hard HEAD

# Или откати к бэкапу
git checkout backup-before-ai-optimization
```

**Всегда есть бэкап в ветке `backup-before-ai-optimization`!**
